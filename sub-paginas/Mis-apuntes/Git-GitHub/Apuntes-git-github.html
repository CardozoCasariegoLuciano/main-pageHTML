Mis apuntes de Git y GitHub

Que es git?

Git, es un software de control de versiones diseñado por Linus Torvalds.

¿Que es un controlador de versiones?
se define como control de versiones a la gestión de los diversos cambios que se realizan sobre los elementos de algún producto o una configuración del mismo.

En pocas palabras Git no trabaja directamente con los archivos, sino con los distintos cambios que nosotros realizamos sobre estos con el paso del tiempo

Manejo de Git

Si estamos en Linux tenemos que posicionarnos dentro de la carpeta en la que queremos trabajar y ahi usar la terminal

Si estamos en Windows Tenemos que abir la terminal "GitBash" que nos viene a la hora de instalar Git dentro de la carpeta en la que vamos a trabajar

Crear un repositorio

git init Con este comando creamos un repositorio en la carpeta en la que estemos parados para comenzar a trabajar con Git

Configuracion de usuario

git config --global user.email "tu@email.com" Indica a git el mail del usuario que realizara los cambios

git config --global user.name "Tu Nombre" Indica a git el nombre del usuario que realizara los cambios

git config --list podremos ver toda la configuraciones, por ahi tiene que estar el mail y nombre del usuario (si no estan es porque todavia no tiene nada configurado)

Estado de nuestro repositorio

git status Con este comando podemos ver el estado de nuestro repositorio, desde la rama en la que estamos parados hasta los archivos sin stagear o commitear

Staging

El Stagin es un area en memoria Ram, donde vas a guardar todos los cambios antes de enviarlos al repositorio
git add . Agrega al stagin todos los archivos
git add [unArchivo] Agrega al stagin el archivo espesifico

Para remover un elemento del area de Stagin puedo usar el comando
git reset HEAD

Commit

Una vez tengo mis nuevos cambios en el area de staging, los tengo que mandar al repositorio:
git commit -m "unMensaje" el atributo -m " " es obligatorio, en el mensaje tengo que describir los cambios realizados en pocas palabras

git commit -am "unMensaje" agregando como atribulo -a nos ahorramos tener que hacer un git add a ese cambio, pero solo para aquillos archivos que ya fueron agregados al stagin alguna vez en su historia

git commit --amend -m "unMensaje" lo que hace el --amend es "fucionar" ese comit con el anterior. Util cuando olvidas incluir algo en ese commit anterior

Deshacer un cambio

git restore [nombre-del-archivo] vulve a la ultima vercion del archivo y lo quita incluso de la lista "no stage" (deshace todos los cambios sobre el archivo desde el ultimo commit)

Eliminar del repositorio

git rm --force [nombre-del-archivo] elimina el archivo del repositorio

Ver la historia de un archivo

git log [nombre-del-archivo] Muestra todos los commit que afectaron al archivo seleccionado, junto al texto seleccionado y su codigo de identificacion

git log (sin nada) muestra todos los cambios del repositorio

git log [sin o con archivo] --stat te muestra ademas de la historia, cuantos cambios se hicieron

git log --all te muestra TODA la historia

git log --all --graph te muestra una especie de grafico (puede ir sin el --all) sobre las ramas

git log --all --graph --decorate --oneline muestra lo mismo que el anterior, pero mas comprimido y facil de leer

git reflog muestra lo mismo que log, pero absolutamente todo, incluso las cosas que fueron eliminadas. Es especialmente util cuando tenemos que recuperar una vercion ya eliminada

Diferencia del ultimo commit

git show Muestra las diferencias del ultimo commit con su vercion anterior

git show [CodigoDeUnCommit] Muestra las diferencias de ese commit con su vercion anterior (el cogido puedo sacar con un log)

Diferencias entre verciones

git diff Muestra las diferencias de mi trabajo actual con la ultima vercion que tenga en el repositorio

git diff --staged Muestra las diferencias de lo que tenga en el staging area (add) con la ultima vercion que tenga en el repositorio

git diff [CodigoDeUnCommit] [OtroCodigoDeUnCommit] Muestra las diferencias entre dos verciones distintas (el cogido puedo sacar con un log)

Volver a una vercion anterior

git reset [IdentificadorDeLaVercionQueQueremos] --hard "Vuelve en el tiempo" y borra las verciones posteriores a esa que elegimos (NO se usa a la ligera, vas a perder todo el travajo de las verciones borradas)

git reset [IdentificadorDeLaVercionQueQueremos] --soft "Vuelve en el tiempo" pero lo que este en el staging sigue ahi (No es muy usado)

Ir a otra vercion

git checkout [IdentificadorDeLaVercionQueQueremos] Nos muestra como era todo nuestro repositorio al momento de esa vercion. Util para ver como eran las cosas en otras verciones.

git checkout [IdentificadorDeLaVercionQueQueremos] [unArchivo] Solo nos muestra como era nuestro archivo elegido al momento de esa vercion.

git checkout master Vuelve a la ultima vercion del repocitorio

git checkout -b [unNombre] ya estando dentro de una vercion antigua, este comando nos crea una rama nueva (ver al lado) a partir de esa vercion

Ramas/Branches

Se trata de una bifurcación del estado del código que crea un nuevo camino de cara a la evolución del código, en paralelo a otras ramas que se puedan generar.

git branch [ElNombreDeLaRama] Para crear una rama nueva

git checkout [ElNombreDeLaRama] Para moverme a otra rama (para volver al master es: git checkout master)

git branch Para ver todas las ramas

git branch -D [NombreDeLaRama] Pare eliminar una rama

Merge: Unir dos ramas

un Merge es unir dos ramas. los cambios van a terminar en la rama en la que este parado a la hora de hacer un merge.

git merge [ElnombreDeLaRamaQueQuieroMergear] -m "unComentario" Realiza el merge con la rama elegida

En caso de conflicto (que dos ramas modifiquen las mismas partes del archivo) lo tengo que solucionar a mano, elijiendo la vercion de la rama que yo quiera y luego hago el Commit para guardar los cambios

git merge --abort Para abortar la fucion

Repositorios remotos

Que es un repositorio remoto?
Los repositorios remotos son versiones de tu proyecto que están hospedadas en Internet o en cualquier otra red

git clone [URL del repositorio en remoto] Simplemente baja a la computadora el repositorio mio o de otra persona.

git remote add [origin] [URL de mi repositorio en remoto] Relaciona nuestro repositorio local con el remoto (de GitHub u otros).
Lo mas comun y estandar es usar el nombre [origin] para referirce al repocitorio remoto, pero puede ser cualquier nombre

git remote set-url [origin] [nuevo URL] Cambia el repositorio remoto en el que esta asociado mi repositorio local

git remote -v Nos muestra si tenemos un repocitorio remoto asociado, y si es asi nos muestra cual es para bajar (fetch) o subir (push) cambios. Generalmente es igual en ambos casos


git pull [origin] master Para descargar cosas del repositorio remoto(Y actualizar con los nuevos cambios)

git push [origin] master Para Subir cosas al repositorio remoto

IMPORTANTE!! este es un caso muy puntual: supongamos que los archivos que estan en mi repositorio local y remoto son distintos. Si lo que yo quiero es subir mis archivos del repositorio local al repositorio remoto no me va a dejar ya que las historias de esos arichos no esta relacionada, primero tengo que traer lo que este en el remoto a mi repositorio local con el comando:

git pull origin master --allow-unrelated-histories Comando que trae la versión del repositorio remoto y hacer merge para crear un commit con los archivos de ambas partes.

una vez ya tenga todo lo del repositorio remoto en mi local, ya voy a poder subir y bajar cosas sin problemas

Si desde un principio mi repositorio local esta vacio, no hace falta, solo con un gil pull alcanza

LLaves publicas y pribadas/SHH

Las llaves SSH son una forma de proteger nuestro codigo con una capa de encriptacion, son mucho mas seguras que el HTTPS.
Notas:
   Podemos clonar nustro repositorio con SSH en lugar de HTTPS
   No es necesario poner mi usuario y contraseña todo el tiempo
   Es una llave para cada equipo(computadora), no por pesona

Pasos a seguir para configurar una clave SHH en Linux:
   Nota: las claves SSH no son propias de Git, pueden ser usadas en otras aplicasiones

Paso 1) ssh-keygen -t rsa -b 4096 -C "tu@email.com"
   Esto va a crear una clave SSH en nuestra computadora.
   nos va a preguntar donde queremos guardarla, pero la ubicacion por defecto esta bien, solo le damos enter otra vez.
   Luego nos va a preguntar por una contraseña estra para esa llave, no es nesesaria (puedo darle enter) pero esta bueno tenerla

Paso2) eval $(ssh-agent -s)
  Enciende el "servidor" de llaves SSH de tu computadora
   nos muestra algo como "agent pid unNumero" indicando que esta todo bien

Paso3) ssh-add ruta-donde-guardaste-tu-llave-privada
  Añade tu llave SSH a este "servidor":
   la ruta por defecto tendria que ser: ~/.ssh/id_rsa

Paso4) copiar el contenido del archivo id_rsa.pub que esta en la carpeta ~/.ssh


Paso5) Ir a GitHub > settings de nuestro perfil > en la seccion "SHH and GPG keys" > clic en "new clave SSH" > Elijo un titulo y pego el contenido del punto 4
   hasta aca ya tengo mi cuenta de git asociada con la clave SSH de mi PC, pero todabia no la estoy usando

Paso6) git remote set-url [origin] [miRepositorio con SHH]
   en mi repositorio remoto busco y copio el codigo para clonar con SHH y lo pego en ese comando

Y listo, ya tengo una clave SSH en mi computadora conectada a GitHub

Alias de Git

Un alias es una abreviacion de un comando (generalmente uno largo) para tener acceso de forma mas rapida.
Los alias "normales" son propio de linux, no de git, y estos se borran al cerrar la terminal.
Los alias de git no se borran

git config --global alias.ElnombreDelAlias "Nombre del comando" crea un alias

git "elnombreDelAlias" accede a ese alias

Manejo de ramas

git push origin [LaRama] Para mandar una rama al repositorio remoto

git pull origin [LaRama] Traer una rama desde el repositorio remoto

git push origin --delete [LaRama] Borrar ramas del repositorio remoto que ya no esten en el local

git branch Para ver las ramas locales

git branch -r Para ver las ramas remotas

git branch -a Para ver todas las ramas

Git ignore

Con el git ignore nos aseguramos que git va a ignorar siertos archivos a la hora se subirlos al repositorio remoto, como por ejemplo archivos bianrios, archivos que tengan nuestras contraseñas, etc.
Para hacerlo tenemos que crear un archivo llamado .gitignore y dentro poner los archivos que vamos a ignorar (uno por renglon)

por ejemplo: si pongo [*.jpg] git va a ignorar todos los archivos .jpg que quiera subir

GitHub Pages

GitHub pages es un servicio de hosting gratuito que ofrece github para publicar tu pagina web Esta bueno para crear tu portafolio

Pasos:
1) Estando en GitHub crear un nuevo repositorio publico llamado [TuNombreDeUsuario].github.io
2) Clonar ese repositorio vacio al home de tu PC y ahi dentro crear un index.html
3) Modificar ese index a gusto, hacerle el git add, commit, pull y push
4) Listo, ahora ya tengo en internet mi pagina web, para buscarca en el navegador el url es: [TuNombreDeUsuario.github.io]

Si te tira error podes intentar con:
5) Estando al repocitorio en GitHub ir a "Setings" y buscar la seccion "GitHub Pages", ahi en el campo Source cambiar el "None" por "Master branch" una vez cargado eso te va a decir la url de tu pagina

Git stash

Es una forma útil de tener en temporal los cambios para poder moverse entre ramas y luego poder recuperar esos cambios. Cuando hago un cambio por error y no quiero hacerle un comit uso el git stash

git stash Guarda los cambios en un lugar temporal y vuelve al estado anterior

git stash list Muestra la lista de todos mis stash (WIP = work in progres)

git stash pop Devuelve los cambios guardados en stash (se tiene que hacer en la misma rama en la que se hizo el stash)

git stash branch [NombreDeNuevaRama] Mueve ese stash a una nueva rama (el stash ya tiene que estar creado)

git stash drop Borra el ultimo stash

Busquedas

los siguientes comandos son para ver cuantas veces y/o donde use determinada palabra, tanto en tus archivos como en commits

git grep [palabraAbuscar] busca la palabra en todo el repositorio y en que archivo se encuentra

git grep -n [palabraAbuscar] ademas muestra en que linea se encuetra

git grep -c [palabraAbuscar] muestra cuantas veces aparece esa palabra

git log -S "[palabraAbuscar]" muestra todos los commits en los que se menciona o tiene que ver la palabra elegida
